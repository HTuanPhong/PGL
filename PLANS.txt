# Lessons:

do not compare float to 0 please!!! floating error !! if num != 0 do this if num > 1e-10 or num < -1e^-10

arena allocator for frame.

the reason memory leak is because a programe have a long loop

this arena can be pass through every functions called within a frame to carry error messages or datas.

prefer passing value down the call instead of global because we have a clear scope.

passing arena down the call chain by copy and by ptr is a cool way to have RAII (basically just not have to call free)

try to have ZII (basically to avoid calling init)

immediate mode is faster when we need to change the structure frequently like 3D world and game GUI.
retained mode is faster when we dont change the structure frequently and instead change it in bulk
like gpu pipeline, command buffer, os gui where we cant afford the cost to rebuild while for the above we have to rebuild anyway. 

gui have little thing to draw so these shader branching and single draw call make sense 
but for a 3d world with many object this will slow down and you should do thing seperately instead

"If this code runs a different number of times between two frames, could it change the outcome of the game?"
If the answer is Yes (e.g., applying velocity, checking for a hit), it must go in the Tick. because deterministic is important.
If the answer is No (e.g., updating the position of a sliding menu), it should go in the Draw step. because it will be smoother.
